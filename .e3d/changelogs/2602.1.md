## 2026-03-02 - Closed-loop Excel + Reactive Live Mode Stabilization

  - preserve table context for subsequent deterministic reads/writes.
- Added closed-loop recalc workflow:
  - read current PQ block from Excel
  - run CalcsLive calculation
  - write output values back to mapped rows in value column.
- Added initial output prefill after load by reusing the same recalc pipeline.
- Improved table robustness in `excel-bridge/excel_api.py`:
  - metadata block support
  - header row scanning below ArticleID
  - more reliable auto-detection for populated article layouts.
- Implemented event-driven Live Mode in bridge (Option A):
  - new `excel-bridge/live_watcher.py` using Excel `SheetChange` COM events
  - debounce + recalc trigger + writeback
  - suppression guard to avoid self-trigger loops.
- Added bridge endpoints for watcher control/status:
  - `POST /excel/live-mode/start`
  - `POST /excel/live-mode/stop`
  - `GET /excel/live-mode/status`
- Updated Streamlit dashboard to use bridge-driven Live Mode status/control (no UI-side polling recalc loop).

- Standardized to Bearer token auth only (`Authorization: Bearer <token>`) for CalcsLive API calls.
- Removed legacy `X-API-Key` header and request-body API key fields.

- Prefill and manual recalc work via deterministic closed-loop flow.
- Reactive live recalc now works from Excel cell changes through event-driven watcher.
